# CloudSewa Hosting Plans and Service Scope Policy

## 1. Hosting Plan Categories

CloudSewa offers three major hosting categories:

1. Shared Hosting
2. WordPress Hosting
3. VPS Hosting

Each category has different support scope, resource limits, and technical responsibility.

## 2. Shared Hosting

Shared hosting is suitable for small websites, blogs, portfolios, and low-traffic business websites.

Shared hosting includes cPanel access, website file manager, MySQL database, email accounts depending on plan, free SSL installation support, DNS guidance, basic backup availability, one-click WordPress installation, PHP version selection, and basic malware scan depending on plan.

Shared hosting does not include root access, custom server daemon installation, long-running background processes, crypto mining, high CPU workloads, mass mailing, custom firewall rules, or dedicated resources.

## 3. WordPress Hosting

WordPress hosting is optimized for WordPress websites.

WordPress hosting includes WordPress installation support, SSL installation, basic plugin compatibility guidance, basic caching setup, theme upload support, staging support depending on plan, backup restore support depending on plan, and malware cleanup assistance for managed plans.

WordPress hosting does not include custom theme development, custom plugin development, fixing broken third-party plugin code, SEO service, content writing, custom design service, or WooCommerce business operation support.

## 4. VPS Hosting

VPS hosting gives customers dedicated virtual server resources.

VPS hosting includes virtual CPU allocation, RAM allocation, SSD/NVMe storage, public IP address, root access, OS reinstall option, firewall panel, reboot option, bandwidth allocation, and optional control panel installation.

CloudSewa offers unmanaged and managed VPS.

## 5. Unmanaged VPS Scope

In unmanaged VPS, CloudSewa is responsible for VPS hardware availability, network availability, virtualization platform, power and datacenter-level availability, VPS provisioning, VPS restart/rebuild from panel, and basic connectivity investigation.

Customer is responsible for operating system configuration, web server setup, database setup, application deployment, firewall configuration, security patching, SSH key management, software updates, backups, malware cleanup, and application performance tuning.

Support for unmanaged VPS is limited to infrastructure-level issues.

## 6. Managed VPS Scope

Managed VPS includes additional support.

CloudSewa may help with initial server hardening, web server installation, control panel installation, SSL setup, basic firewall configuration, server monitoring, basic performance investigation, backup configuration, security patching, and incident response guidance.

Managed VPS does not include custom application debugging, full DevOps engineering, custom CI/CD pipeline setup, complex database optimization, application code fixes, third-party API debugging, business logic issue solving, or unlimited emergency engineering work.

## 7. Resource Fair Usage

Customers must not use CloudSewa hosting for crypto mining, botnets, spam sending, phishing pages, malware hosting, copyright abuse, open proxy services, DDoS attack tools, resource abuse affecting other customers, or illegal content.

Shared hosting customers must follow fair usage limits. If a shared hosting account consumes excessive CPU, memory, I/O, or email resources, CloudSewa may temporarily limit the account and recommend upgrade.

## 8. Plan Upgrade Rules

Customers may upgrade hosting plans at any time.

Upgrade rules:

1. Upgrade is usually instant after payment.
2. Existing files remain unchanged during normal upgrade.
3. Resource increase may require service reload.
4. VPS upgrade may require reboot.
5. Downgrade may not be possible if current usage exceeds lower plan limit.
6. Billing is adjusted based on remaining service period.

## 9. Plan Downgrade Rules

Downgrade is allowed only when current storage usage fits lower plan, email account count fits lower plan, database count fits lower plan, bandwidth usage is within lower plan limit, and customer confirms feature reduction.

Downgrade may be scheduled for next billing cycle instead of immediate change.
