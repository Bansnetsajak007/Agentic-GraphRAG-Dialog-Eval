# CloudSewa Abuse and Security Policy

## 1. Abuse Policy Overview

CloudSewa does not allow customers to use services for harmful, illegal, or abusive activity.

Abuse reports are reviewed by the security and abuse team.

## 2. Prohibited Activities

CloudSewa services must not be used for phishing, malware hosting, botnet control, spam sending, credential stealing, crypto mining without explicit approval, DDoS attacks, port scanning at scale, illegal content distribution, child exploitation material, copyright infringement, fraudulent websites, fake banking or wallet pages, open proxy abuse, command-and-control infrastructure, or unauthorized vulnerability scanning of third parties.

## 3. Malware Infection

If customer website is infected with malware, CloudSewa may notify customer, temporarily disable malicious files, suspend website if risk is high, recommend cleanup, restore from clean backup if available, or require security patching before reactivation.

For unmanaged VPS, malware cleanup is customer responsibility unless managed support is purchased.

## 4. Spam Abuse

If service is used for spam, CloudSewa may suspend outgoing email, limit mail sending, suspend hosting account, ask customer to remove compromised script, ask customer to update password, require SPF/DKIM/DMARC correction, or terminate service for severe abuse.

## 5. Phishing Complaint

Phishing complaint is treated as critical.

CloudSewa may suspend the website immediately while investigating.

Customer must provide explanation and remove phishing content before service is restored.

Repeated phishing abuse may lead to termination.

## 6. DDoS Attack

If customer service is under DDoS attack, CloudSewa may apply traffic filtering, null-route IP temporarily, move service behind protection if available, recommend DDoS protection add-on, or suspend service if it threatens network stability.

Basic hosting plans may not include advanced DDoS protection.

## 7. Compromised Account

If customer account is compromised, support may require identity verification, password reset, 2FA enablement, email verification, review of recent changes, malware scan, API key rotation, and SSH key review.

## 8. Security Responsibility

CloudSewa is responsible for infrastructure security, hypervisor patching, network security controls, platform-level monitoring, shared hosting server patching, and control panel security updates.

Customer is responsible for strong passwords, application updates, plugin updates, theme updates, secure code, SSH key security, admin account protection, website backup, avoiding nulled themes/plugins, and removing malware from unmanaged services.

## 9. Abuse Appeal

Customer may appeal suspension by submitting service ID, explanation, actions taken, security cleanup proof, confirmation that abuse will not repeat, and request for temporary access if needed.

CloudSewa may deny appeal for severe or repeated abuse.
